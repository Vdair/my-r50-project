# 🚀 快速修复 405 错误

## 问题

您遇到的错误：
```
status: 405
error: "Method Not Allowed"
path: "/api/coze/run"
```

**原因**：缓存问题 - 应用仍在使用旧的代理路径。

---

## ⚡ 快速解决方案

### 方案 1：强制清除缓存（3 步搞定）

```bash
# 1. 停止开发服务器（Ctrl + C）

# 2. 清除所有缓存
cd /workspace/app-8htx34d81fcx
./force-clear-cache.sh

# 3. 重启开发服务器
npm run dev:h5
```

**然后在浏览器中：**
1. 打开**无痕窗口**（Ctrl + Shift + N）
2. 访问 `http://localhost:10086`
3. 测试参数生成

---

### 方案 2：使用独立 Web 应用（最简单）

```bash
cd /workspace/app-8htx34d81fcx/standalone-web
python3 -m http.server 8000
```

在浏览器中打开：`http://localhost:8000`

**优势**：
- ✅ 无缓存问题
- ✅ 无代理问题
- ✅ 立即可用

---

## 🔍 如何验证修复成功？

打开浏览器开发者工具（F12）：

### 1. 检查 Console 标签

应该看到：
```
🔗 使用完整 URL: https://3mp9d3y2dz.coze.site/run
```

### 2. 检查 Network 标签

点击"生成最佳参数"后，请求 URL 应该是：
- ✅ `https://3mp9d3y2dz.coze.site/run`
- ❌ 不是 `/api/coze/run`
